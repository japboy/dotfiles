# AI-Like Japanese Patterns

These patterns are warning signs, not prohibited expressions. Inspect them when
the text feels generic, literal, or overly polished. Keep them when they are the
most accurate or domain-appropriate wording.

## Common Warning Signs

| Pattern | Risk | Possible alternatives |
|---|---|---|
| 〜において | Stiff translationese when a simple particle works | 〜で, 〜では, 〜の場合 |
| 〜することができます | Overly indirect ability expression | 〜できます, 〜られます, a simpler predicate |
| 〜を行う | Verb nominalization that weakens the sentence | 実行する, 設定する, 確認する, 使う |
| 〜を実施する | Bureaucratic or process-heavy tone | 行う, 試す, 導入する, 実行する |
| 包括的な | Generic and often unnecessary | 全体的な, 幅広い, 必要な範囲を含む |
| 多岐にわたる | Vague breadth claim | 具体的な category list, 幅広い |
| 重要な役割を果たす | Stock phrase | 重要です, 支えます, 要になります, 担います |
| 〜を促進する | Abstract business wording | 進める, 後押しする, しやすくする |
| 〜を活用することで | Inflated means clause | 〜を使うと, 〜により, 〜で |
| 本質的には | Unnecessary framing | Usually delete, or replace with 要するに |
| 文脈に応じて異なります | True but often evasive | Specify the actual decision factors |
| 以下に示します | Template-like transition | Remove, or use 例を挙げると |
| 〜という観点から | Abstract frame | 〜では, 〜を見ると, 〜の面では |
| 〜に対して有効です | Stiff evaluation phrase | 〜に効きます, 〜で役立ちます, 〜に向いています |

## Before and After Examples

### Product Context

Source:

```text
ユーザーは、自身のニーズに応じて機能を構成することができます。
```

Better:

```text
ユーザーは、自分の用途に合わせてこの機能を設定できます。
```

### Technical Context

Source:

```text
消費者はトピックからメッセージを読み取ります。
```

Better:

```text
コンシューマーはトピックからメッセージを読み取ります。
```

### Stiff Explanation

Source:

```text
この関数は、データ処理の文脈において重要な役割を果たします。
```

Better:

```text
この関数は、データ処理で重要な役割を担います。
```

More concise, if the context allows:

```text
この関数は、データ処理の要になります。
```

## Caution

Do not replace a pattern only because it appears in this file. A rewrite is an
improvement only when it better fits the inferred audience, domain, and channel
without changing the meaning.
