# Japanese Style Defaults

Use these defaults when the target context has no project style guide, brand
voice, publication standard, or user instruction that says otherwise.

These are lightweight style preferences, not correctness rules. Apply them only
when they make the Japanese more natural for the inferred audience and channel.

## Orthography Defaults

- Use できる, not 出来る, for auxiliary usage.
- Use ください, not 下さい, for auxiliary usage.
- Use こと, not 事, for abstract nominalization.
- Use とき, not 時, for general temporal clauses.
- Use いただく, not 頂く, for auxiliary usage.
- Use さま instead of 様 only when the product's tone guide explicitly prefers
  it.

## Application Rules

1. Preserve user-provided style guide requirements.
2. Preserve official product, legal, citation, and quoted text unless the user
   explicitly asks to normalize it.
3. Do not change identifiers, code, file paths, commands, URLs, or API names.
4. Do not apply these defaults mechanically inside terminology, titles, or
   proper nouns.
5. When naturalness and consistency conflict, prefer the target document's
   existing style unless it is clearly accidental.
